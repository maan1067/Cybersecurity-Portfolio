
# Elastic Stack: The Basics

## Overview

Elastic Stack, commonly known as the **ELK Stack**, is an open-source platform used for collecting, storing, searching, analyzing, and visualizing large volumes of log and event data.

It is widely used in Security Operations Centers (SOCs) for centralized log management, threat detection, incident investigation, and security monitoring.

The Elastic Stack consists of four core components:

- Elasticsearch
- Logstash
- Kibana
- Beats

Together, these components provide an end-to-end solution for log collection and analysis.

# What is Elastic Stack?

Elastic Stack is a collection of open-source tools that work together to collect, process, store, search, and visualize machine-generated data.

The platform is designed for:

- Security Monitoring
- Threat Hunting
- Incident Response
- Log Management
- Infrastructure Monitoring
- Application Monitoring

# Elasticsearch

Elasticsearch is the search and analytics engine.

It stores indexed data and allows analysts to perform fast searches across millions of events.

### Responsibilities

- Store logs
- Index documents
- Execute searches
- Aggregate data
- Support real-time analytics

---

# Logstash

Logstash is the data processing pipeline.

It receives data from multiple sources, transforms it if needed, and forwards it to Elasticsearch.

### Responsibilities

- Collect logs
- Parse data
- Filter events
- Transform fields
- Forward processed logs

---

# Kibana

Kibana is the visualization and investigation interface.

SOC analysts use Kibana to:

- Search logs
- Create dashboards
- Build visualizations
- Investigate alerts
- Monitor environments

---

# Beats

Beats are lightweight agents installed on endpoints.

Different Beats collect different types of data.

Examples include:

- Filebeat
- Winlogbeat
- Packetbeat
- Metricbeat
- Auditbeat
- Heartbeat

---

# Data Flow

The normal Elastic workflow is:

1. Data is generated by endpoints.
2. Beats collect the logs.
3. Logstash processes the data.
4. Elasticsearch stores and indexes it.
5. Kibana allows analysts to search and visualize the data.

# Splunk vs Elastic

| Feature | Splunk | Elastic |
|----------|---------|----------|
| License | Commercial | Open Source + Commercial |
| Query Language | SPL | KQL / Lucene |
| Visualization | Dashboards | Kibana |
| Data Storage | Proprietary | Elasticsearch |
| Popular Usage | Enterprise SOC | Enterprise & Open Source SOC |
