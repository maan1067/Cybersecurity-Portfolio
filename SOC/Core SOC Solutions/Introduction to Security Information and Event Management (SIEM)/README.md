# Introduction to Security Information and Event Management (SIEM)

## Overview

Security Information and Event Management (SIEM) is a centralized security solution that collects, stores, normalizes, analyzes, and correlates security logs from multiple devices, applications, and network infrastructure. It enables Security Operations Center (SOC) analysts to monitor environments in real time, investigate security incidents, detect threats, and support incident response activities.

This lab introduces the fundamentals of SIEM, its architecture, core components, and how it supports modern Security Operations Centers.

# What is SIEM?

Security Information and Event Management (SIEM) is a centralized platform used to collect, normalize, analyze, and correlate security events generated by multiple systems across an organization.

Instead of manually reviewing logs from every individual device, SOC analysts use SIEM to monitor the entire environment from one centralized console.

---

# Why SIEM is Important

Modern organizations generate millions of events every day.

Examples include:

- Windows Event Logs
- Linux Syslog
- Firewall Logs
- VPN Logs
- DNS Logs
- Active Directory Logs
- Web Server Logs
- Endpoint Security Logs
- Cloud Security Logs

Without SIEM, analyzing these logs manually would be extremely difficult and time-consuming.

# SIEM Architecture

A typical SIEM solution consists of the following components:

## Log Collection

Logs are collected from various sources throughout the organization's infrastructure.

Common sources include:

- Endpoints
- Servers
- Firewalls
- IDS/IPS
- Applications
- Cloud Services
- Authentication Servers
- Network Devices

---

## Log Normalization

Since each system produces logs in different formats, SIEM converts them into a standardized structure.

This makes searching and correlation much easier.

---

## Log Storage

Collected events are securely stored for:

- Incident investigations
- Threat hunting
- Compliance
- Historical analysis

---

## Correlation Engine

The correlation engine analyzes multiple events together to identify suspicious behavior.

For example:

- Multiple failed logins
- Successful login
- Privilege escalation

When these events occur together, they may indicate a brute-force attack.

---

## Alert Generation

If correlation rules identify malicious activity, the SIEM automatically generates alerts for SOC analysts.

Alerts are usually categorized by severity, such as:

- Informational
- Low
- Medium
- High
- Critical

---

## Dashboards & Reporting

SIEM platforms provide dashboards that allow analysts to:

- Monitor security events
- Track KPIs
- Visualize attack trends
- Generate management reports

---

# SIEM Investigation Workflow

A typical investigation follows these steps:

1. Collect logs from multiple sources.
2. Normalize event data.
3. Correlate related events.
4. Generate security alerts.
5. Investigate suspicious activity.
6. Validate Indicators of Compromise (IOCs).
7. Contain and remediate the incident.
8. Document the investigation.

---

# Example Investigation

An attacker attempts multiple failed logins against a Windows server.

Shortly afterward:

- A successful login occurs.
- PowerShell executes.
- A new administrator account is created.
- The system connects to an unknown external IP.

The SIEM correlates these events and generates a High Severity Alert.

The SOC analyst investigates the timeline, validates the attack, and initiates incident response procedures.

# Real-World Scenario

### Scenario

An employee account generates multiple failed login attempts followed by a successful login from an unfamiliar geographic location.

Minutes later, PowerShell is executed and a scheduled task is created.

### Detection

The SIEM correlates authentication logs, process creation events, and network activity to generate a High Severity Alert.

### Investigation

The SOC analyst reviews:

- Authentication logs
- Windows Event Logs
- Process execution history
- Network connections
- User activity timeline

### Response

- Validate the alert.
- Disable the compromised account.
- Isolate the affected endpoint.
- Block malicious IP addresses.
- Document the incident.
